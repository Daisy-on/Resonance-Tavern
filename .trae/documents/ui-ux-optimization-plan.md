# UI/UX 视觉优化计划

## Summary
优化调酒页面的 UI 布局，解决 HUD 遮挡工具的问题，并提升信息展示的直觉性。

## Current State Analysis
1.  **HUD 遮挡**：右下角的 `actions-panel` 在 `mixing_view` 模式下虽然隐藏了大部分按钮，但其容器仍然存在，且包含“重做”和“上酒”按钮，遮挡了后期的调酒工具。
2.  **信息冗余**：左上角 `status-panel` 包含过多实时参数（振幅、周期等），导致面板过大，遮挡示波器边缘。
3.  **布局散乱**：特殊事件和客人喜好提示目前分布在示波器上方，不够集中。
4.  **未解锁清单**：目前作为一个独立的右上角面板，占用空间。

## Proposed Changes

### 1. `src/ui/hud-renderer.ts`
*   **左上角 HUD 抽屉化**：
    *   修改 `status-panel` 的 HTML 结构，增加一个切换按钮（例如 `[+] 状态`）。
    *   将详细状态信息和“未解锁道具清单”放入抽屉容器中。
    *   移除“当前参数”和“高阶属性”的文本显示（移至画布渲染）。
*   **拆分底部按钮**：
    *   创建一个新的 `table-controls` 容器，用于放置“重做”、“提交上酒”和“档案室”按钮。
    *   修改 `actions-panel` 的渲染逻辑，在 `mixing_view` 下彻底隐藏或移除其原有结构。

### 2. `src/styles.css`
*   **抽屉样式**：
    *   为 `status-panel` 增加过渡动画，实现展开/收起效果。
    *   设置抽屉在收起时仅显示标题或切换按钮。
*   **按钮新布局**：
    *   设置 `table-controls` 的样式，使其固定在桌子边缘线上方（约 `bottom: 310px` 左右）。
    *   移除 `locked-panel` 的独立样式。
    *   调整按钮的排列方式。

### 3. `src/ui/bar-renderer.ts`
*   **示波器信息重排**：
    *   在 `drawMixingFocusView` 中，获取当前 `DrinkState`。
    *   在示波器大屏的右上角渲染“当前参数”和“高阶属性”。
    *   将 `state.activeEvent`（特殊事件）和 `state.currentOrder.moodText`（客人喜好）的绘制位置移动到示波器屏幕底部的空白区域。
    *   调整调酒提示（Hints）的位置，使其位于上述信息下方。

## Assumptions & Decisions
*   **抽屉交互**：默认收起状态，点击切换。
*   **坐标计算**：桌子边缘线在 `mixing_view` 中位于 `h - 300`，因此按钮应放在 `h - 310` 附近。
*   **颜色保持**：继续沿用现有的赛博朋克配色方案。

## Verification Steps
1.  进入调酒特写页面 (`mixing_view`)。
2.  确认左上角 HUD 仅显示切换按钮，点击可展开看到资金、电力、评分及未解锁清单。
3.  确认右下角不再有遮挡工具的面板。
4.  确认“重做”、“提交上酒”、“档案室”按钮整齐排列在桌子上方。
5.  确认示波器大屏右上角实时更新波形参数。
6.  确认特殊事件和客人喜好提示已移至屏幕底部区域。
7.  检查不同分辨率下的适配情况。
