# 道具映射与搅拌逻辑纠偏计划

## Summary
本计划用于修复“道具影响不纯、搅拌行为越权、命名不一致”三类问题，目标是让每个道具只影响其设计维度，并让相位调整操作更省力、更直观。

本次已锁定的产品决策：
- 将“稳频剂”改名为“捣拌棒”，功能保持“减频/拉长周期”不变。
- 搅拌拆分为两个动作：顺时针搅拌（相位+）与逆时针搅拌（相位-）。
- 苏打水严格只影响 `sparkle`，不再影响任何其他属性（包括体积）。

## Current State Analysis
- 当前动作定义仅有单一 `stir`，未区分方向：`src/systems/mixology/mix-actions.ts`。
- `stir` 在实现中同时修改 `phaseOffset`、`blend`、`sparkle`，会间接改变波形纹理/形态：`src/systems/mixology/mixology-system.ts`。
- `wave-generator` 将 `blend/oxidation` 合成为 `harmonics`，因此任何对 `blend` 的改动都会改变波形形状细节：`src/systems/wave/wave-generator.ts`。
- “稳频剂”当前数据 id 为 `tonic_essence`，名称和文案仍是“稳频剂”：`src/content/ingredients.ts`、`src/ui/hud-renderer.ts`、`src/ui/bar-renderer.ts`。
- 苏打水当前会影响 `strength` 与 `volume`，不符合“仅增加毛刺”的期望：`src/content/ingredients.ts`。
- 拖拽命中与按钮启用映射仍以旧动作集合为主，需要扩展到顺/逆搅拌：`src/game/game-loop.ts`、`src/ui/hud-renderer.ts`、`src/ui/bar-renderer.ts`。

## Proposed Changes

### 1) 统一“捣拌棒”命名（不改功能）
- 文件：`src/content/ingredients.ts`
  - 将 `tonic_essence.name` 从“稳频剂”改为“捣拌棒”。
  - 描述同步改为“用于降低频率并拉长周期”。
- 文件：`src/ui/hud-renderer.ts`
  - `add_tonic` 按钮文案改为“捣拌棒 (周期+)”。
- 文件：`src/ui/bar-renderer.ts`
  - 右侧道具标签与拖拽预览文案由“稳频剂”改为“捣拌棒”。
- 文件：`src/ui/pixel-assets.ts`
  - 仅在注释/语义上改名（可保持 sprite key 为 `tonic_vial`，避免触发无谓存档兼容问题）。

### 2) 搅拌拆分为顺/逆两个动作，并严格只改相位
- 文件：`src/systems/mixology/mix-actions.ts`
  - 新增 `stir_cw`、`stir_ccw`，保留或移除 `stir`（计划采用保留兼容，内部映射到 `stir_cw`，避免旧存档动作历史崩溃）。
- 文件：`src/systems/mixology/mixology-system.ts`
  - `stir_cw`: 仅 `phaseOffset += PHASE_STEP`。
  - `stir_ccw`: 仅 `phaseOffset -= PHASE_STEP`。
  - 删除 `stir` 对 `blend/sparkle` 的任何修改。
  - 删除 `bitters` 对 `phaseOffset` 的隐式改动，确保“道具影响不越权”。
  - `shake/pour_precise` 不再改相位，只保留其各自定义维度（如纹理/混合相关），防止相位被多道具串扰。

### 3) 道具影响矩阵纠偏（消除额外影响）
- 文件：`src/content/ingredients.ts`
  - `soda_water` 改为严格只改 `sparkle`（`strength/sweetness/acidity/temperature/volume` 均置 0）。
  - 其余添加剂按“单一职责”梳理：
    - 糖浆：只改 `sweetness`
    - 柠檬：只改 `acidity`
    - 冰块：只改 `temperature`（若需保留稀释，明确写入文档并在 HUD 显示）
    - 苦精：仅保留既定“微调维度”，不改相位
    - 捣拌棒（旧 `tonic_essence`）：只改 `sweetness`（负向）
- 文件：`src/systems/mixology/mixology-system.ts`
  - 移除配料 id 的硬编码特例副作用（如 bitters 对 phaseOffset、syrup 对 blend），改为纯数据驱动。

### 4) 交互与可视化接入顺/逆搅拌
- 文件：`src/ui/hud-renderer.ts`
  - 新增两个按钮：`顺时针搅拌 (相位+)`、`逆时针搅拌 (相位-)`。
  - `actionUnlockMap` 映射两个动作到同一解锁项 `stir_tool`。
- 文件：`src/ui/bar-renderer.ts`
  - 在调酒台右侧绘制两根搅拌棒（或同一棒左右两个命中区），文字区分顺/逆。
  - 拖拽预览文案新增 `stir_cw` 与 `stir_ccw`。
- 文件：`src/game/game-loop.ts`
  - `mixActions`、`actionUnlockMap`、`canDragAction`、鼠标命中区域分发统一接入 `stir_cw/stir_ccw`。
  - 保留对旧 `stir` 的兼容分支（映射为 `stir_cw`），避免旧输入路径失效。

### 5) 波形映射边界校准（防止“搅拌看起来改形状”）
- 文件：`src/systems/wave/wave-generator.ts`
  - 保持 `phase` 仅来自 `drink.phaseOffset`。
  - `harmonics` 继续来自 `blend/oxidation`，但由于搅拌动作不再触碰这两个值，视觉上将只表现为水平平移，不再“方波被搅成正弦”的错觉。
- 文件：`src/systems/wave/wave-match.ts`
  - 不改评分权重，仅验证相位动作变化后分项（`phase`）响应符合预期，避免本次修复引入评分回归。

## Assumptions & Decisions
- 不改 `tonic_essence` 的数据 id，只改显示名“捣拌棒”，以降低存档/解锁兼容风险。
- “两个搅拌棒”采用两个独立动作入口（按钮与拖拽命中区都分开）。
- 本轮不引入新的平衡系统，仅做“影响关系纠偏 + 交互可达性修复”。
- 旧存档兼容：不提升存档版本号；通过保留旧 action 兼容映射保证读取安全。

## Verification Steps
1. 类型与构建
- 运行 `npm run typecheck`。
- 运行 `npm run build`。

2. 功能验证（调酒行为）
- 仅执行 `stir_cw/stir_ccw`：波形只发生水平位移，`shape/texture` 视觉不变。
- 执行苏打：仅毛刺变化；振幅、频率、衰减、相位不应出现变化。
- 执行捣拌棒（旧稳频剂）：仅周期变长（频率下降），不应改变相位与形态细节。

3. 交互验证（UI 与拖拽）
- HUD 显示“捣拌棒”文案正确。
- 调酒台右侧可分别触发顺时针/逆时针搅拌。
- 拖拽命中与按钮触发结果一致，不出现“点得到但拖不到”。

4. 回归验证（评分与经济）
- 提交订单后 `shape/frequency/phase` 分项变化与操作维度一致（无串扰）。
- 成本扣除仍只发生在道具动作时，且不会因新增搅拌动作重复扣除耗材。
