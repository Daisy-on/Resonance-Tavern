# 吧台视觉优化与 NPC 站位调整方案

## 摘要

本计划旨在提升游戏吧台的视觉表现力，参考《红弦俱乐部》的风格，通过重构吧台几何结构、优化渲染顺序以及微调 NPC 位置，实现更具空间感和沉浸感的“伪 3D”酒吧场景。

## 现状分析

* **吧台结构**：目前的吧台仅为一个扁平的矩形块，缺乏厚度和透视感。

* **空间关系**：酒柜（Shelf）与吧台之间存在较大空隙，未形成“围合”感。

* **NPC 站位**：NPC 绘制在吧台层之上，且坐标偏高，导致看起来像是漂浮在空中或站在一块悬空的板子上。

* **渲染顺序**：当前的绘制顺序是：酒柜 -> 吧台 -> NPC，这违反了“吧台遮挡角色下半身”的常识。

## 提议更改

### 1. 重构吧台几何结构 ([bar-renderer.ts](file:///d:/Homework_vs/game-pr/src/ui/bar-renderer.ts))

* **`drawLShapeTable`** **函数**：

  * **正面 (Front Face)**：绘制一个从 `barTopY` 到屏幕底部的深色矩形，代表吧台的立面。

  * **顶面 (Top Surface)**：绘制一个具有透视角度的平行四边形，颜色稍浅，代表吧台的操作面。

  * **装饰元素**：在吧台边缘添加霓虹灯带，并在正面添加简单的面板装饰线条。

  * **L 型拐角**：优化左侧的透视线条，使其在视觉上能将 NPC “包裹”在内。

### 2. 对齐酒柜与吧台 ([bar-renderer.ts](file:///d:/Homework_vs/game-pr/src/ui/bar-renderer.ts))

* **`drawHighResBarShelf`** **函数**：

  * 动态计算酒柜的高度，使其底部边缘正好与吧台的顶面（Top Surface）重合。

  * 调整酒柜的水平位置，使其位于 L 型吧台的右侧区域。

### 3. 优化渲染流程 ([bar-renderer.ts](file:///d:/Homework_vs/game-pr/src/ui/bar-renderer.ts))

* **`renderBar`** **函数**：

  * 调整执行顺序：

    1. 绘制背景/墙面。
    2. 绘制酒柜。
    3. **绘制 NPC**（此时 NPC 被画在背景上，但会被后续的吧台遮挡）。
    4. **绘制吧台**（利用吧台的正面和顶面实现对 NPC 下半身的自然遮挡）。
    5. 绘制杯子、示波器等交互 UI。

### 4. 微调 NPC 站位 ([bar-renderer.ts](file:///d:/Homework_vs/game-pr/src/ui/bar-renderer.ts))

* 将 NPC 的 `npcY` 坐标下移（例如从 `height - 420` 调整为 `height - 350` 左右），配合新的渲染顺序，使 NPC 看起来是站在吧台后的地面上，且腰部以下被吧台自然遮挡。

## 假设与决策

* **伪 3D 透视**：采用简单的 2.5D 平行透视，不引入复杂的 3D 坐标计算。

* **遮挡逻辑**：完全依赖 Canvas 的绘制顺序来实现遮挡，不使用图层或 Z-index。

## 验证步骤

1. 修改代码后运行 `npm run dev`。
2. 在浏览器中检查：

   * 吧台是否有明显的厚度和顶面。

   * 酒柜底部是否与吧台顶面无缝衔接。

   * NPC 是否看起来“躲”在吧台后面。

   * NPC 的高度是否与环境比例协调，不再有悬空感。
