# 调酒细节深度优化计划

本项目将对“共振酒吧”的调酒核心玩法进行深度重构，引入全新的交互式调酒界面，并支持鼠标拖拽操作。

## 当前状态分析
- 目前调酒逻辑主要依赖 HUD 按钮点击。
- `orderFlow` 状态机控制场景切换，但调酒界面与对话界面共用背景和布局。
- 缺乏鼠标直接操作 Canvas 元素的交互机制。

## 提议的变更

### 1. 游戏状态扩展 (`src/game/game-state.ts`)
- 在 `GameState` 中增加 `mouse` 状态，记录鼠标坐标 `x, y`。
- 增加 `draggedItem` 状态，记录当前正在拖拽的物品（如：基酒瓶、冰块、添加剂瓶）。
- 在 `OrderFlowState` 中增加 `mixing_view` 状态，用于表示专注调酒的特写界面。

### 2. 输入系统升级 (`src/game/game-loop.ts`)
- 为 Canvas 添加 `mousedown`, `mousemove`, `mouseup` 监听器。
- 实现碰撞检测逻辑，识别鼠标是否点击了基酒、冰盒或添加剂。
- 当物品被拖拽到酒杯上方并释放时，通过 `dispatch` 触发对应的 `add_...` 或 `select_...` 动作。

### 3. 全新调酒特写场景 (`src/ui/bar-renderer.ts`)
- 实现 `drawMixingFocusView` 函数，用于绘制全新的调酒特写场景。
- **布局设计**：
    - **背景**：一张巨大的深木色吧台桌面。
    - **左侧**：排列整齐的基酒瓶（伏特加、琴酒、威士忌）。
    - **中间**：专注展示调酒杯和冰块盒。
    - **右侧**：放置糖浆、柠檬、苏打水等添加剂瓶。
- **拖拽反馈**：在渲染循环中，如果存在 `draggedItem`，则在鼠标坐标处绘制该物品的半透明预览。

### 4. 流程与 UI 适配 (`src/ui/hud-renderer.ts`)
- 修改“接受订单”按钮的逻辑，使其跳转至 `mixing_view` 状态。
- 在 `mixing_view` 状态下，隐藏原有的调酒动作按钮面板，改为仅保留“重做”和“提交上酒”按钮。
- 在 `mixing_view` 中显示当前波形示波器，以便玩家观察实时反馈。

### 5. 交互逻辑优化 (`src/systems/mixology/mixology-system.ts`)
- 确保所有的调酒动作能够无缝对接新的拖拽交互。
- 增加冰块拖拽的特殊处理逻辑。

## 验证步骤
1. 点击“开始营业”并“接受订单”，确认页面能够跳转至全新的特写调酒界面。
2. 验证左侧基酒瓶是否可以点击并产生响应。
3. 验证冰块盒是否可以拖出冰块，并能够放置进中间的酒杯。
4. 验证右侧添加剂瓶是否可以拖动到酒杯上方释放并改变波形。
5. 验证提交酒水后，流程是否能正确返回结果展示界面。
