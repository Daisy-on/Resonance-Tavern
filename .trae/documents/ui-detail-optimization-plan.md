# UI 细节进一步优化计划

本计划旨在解决调酒特写页面中提示信息模糊/重叠的问题，并修复左上角 HUD 状态栏标题被遮挡的视觉 Bug。

## 当前问题分析

1. **提示信息位置不佳**：目前的心情文本 (Mood)、事件描述 (Event) 和实时提示 (Hints) 都渲染在示波器屏幕内部的底部。由于屏幕内有扫描线和波形干扰，导致文字看不清且容易与波形重叠。
2. **HUD 标题遮挡**：左上角的 `#status-panel` 在折叠状态下 `max-height` 仅为 `40px`，而 `.panel` 的内边距为 `16px`。这导致内容区域仅剩 `8px` 高度，使得“状态 STATUS”标题被严重截断。

## 拟定修改方案

### 1. 移动示波器提示信息 (bar-renderer.ts)

将渲染位置从屏幕内部移至屏幕外部的正下方。

* **Mood (心情)**：移至屏幕底部边缘下方约 25px 处。

* **Event (事件)**：移至心情文本下方约 20px 处。

* **Hints (提示)**：移至屏幕底部边缘下方约 65px 处，确保与心情/事件文本有足够间距。

* **背景增强**：为这些文本添加半透明黑色背景矩形，以增强在深色背景上的可读性。

### 2. 优化 HUD 样式 (styles.css & hud-renderer.ts)

* **缩减内边距**：将 `.panel` 的通用内边距从 `16px` 调整为 `12px 16px`（上下 12px，左右 16px）。

* **调整折叠高度**：将 `#status-panel` 的折叠 `max-height` 从 `40px` 增加到 `48px`。

* **微调字体与行高**：确保 `#status-toggle` 内部的 `h3` 标题垂直居中，且行高适中。

## 具体修改文件

* `src/ui/bar-renderer.ts`: 修改 `drawMixingFocusView` 函数中的文本渲染坐标。

* `src/styles.css`: 修改 `.panel` 和 `#status-panel` 的相关 CSS 属性。

* `src/ui/hud-renderer.ts`: (视情况) 微调 HTML 结构或内联样式以配合 CSS 修改。

## 验证步骤

1. 进入调酒特写页面。
2. 检查示波器下方的提示信息是否清晰、无重叠，且完全位于绿色屏幕框之外。
3. 检查左上角 HUD 状态栏，确认在折叠状态下，“状态 STATUS”标题显示完整且美观。
4. 展开/收起状态栏，确保动画过渡依然平滑。
