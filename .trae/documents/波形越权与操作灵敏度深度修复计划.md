# 波形越权与操作灵敏度深度修复计划

## Summary
本计划旨在解决用户反馈的两个核心体验问题：
1. **搅拌越权**：搅拌操作在移动波形的同时错误地改变了波形形状（尤其是方波变正弦的错觉）。
2. **操作不灵敏**：顺/逆时针搅拌点击判定范围窄、步长与显示量化不匹配导致的“点了几次没反应”的问题。

## Current State Analysis
### 1. 搅拌改形状的物理根因
- 在 `src/systems/wave/wave-generator.ts` 中，`asymSkew`（酸度产生的非对称变形）的计算使用了包含全局偏移量的 `phase`。
- 公式：`const asymSkew = 1 + asymmetry * Math.sin(phase);`
- 结果：当 `phase` 随搅拌改变时，变形位置相对于波峰/波谷发生了滑动，视觉上表现为形状扭曲。

### 2. 操作“迟钝”的数值根因
- `src/systems/mixology/mixology-system.ts` 中的 `PHASE_STEP` 为 `PI / 16` (≈0.196)。
- `src/systems/wave/wave-generator.ts` 中的相位量化步长为 `2PI / 15` (≈0.418)。
- 结果：点击一次搅拌，相位增量不足以跨越一个量化阶级，导致画面无变化，通常需要连续点 3 次才有反馈。

### 3. 判定不灵敏的坐标根因
- `src/game/game-loop.ts` 中的命中检测盒（Hitbox）起始坐标与 `src/ui/bar-renderer.ts` 中的渲染坐标存在 20 像素左右的系统性偏差。
- 搅拌棒的文字区域位于命中盒的边缘，导致点击文字时经常判定失败。

## Proposed Changes

### 1. 物理公式解耦 (解决越权改形状与放大感)
- **文件**：`src/systems/wave/wave-generator.ts`
- **修改点**：
    - 在 `generateWave` 函数中，计算 `asymSkew` 时使用不带 `q.phase` 的局部相位（即 `t * TAU * q.frequency`）。
    - 检查 `asymmetry` 的计算公式，确保其系数不会导致波形在视觉上产生异常的整体放大（目前 `1 + asymmetry` 最大可达 1.35，这在非对称波形中是正常的，但如果搅拌时这个变形在滑动，会产生“正在变大”的错觉）。
- **预期效果**：搅拌仅产生水平位移，波形的“肥瘦”和“畸变”位置将固定在波形自身坐标系内，消除“动态放大”的错觉。

### 2. 操作步长对齐 (解决反馈迟钝)
- **文件**：`src/systems/mixology/mixology-system.ts`
- **修改点**：将 `PHASE_STEP` 调整为 `(Math.PI * 2) / 16`，确保其与 `PHASE_STEPS` 定义的量化阶梯一一对应。
- **预期效果**：点击一次搅拌按钮，波形立即向左/右跳变一格，提供 100% 的即时正向反馈。

### 3. 布局坐标统一与 Hitbox 扩充 (解决点击不灵敏)
- **文件**：`src/game/game-loop.ts`
- **修改点**：
    - 将搅拌工具的点击检测 Y 轴起始点从 `propY + 40` 下移至 `propY + 50`，并将高度从 `80` 扩展至 `100`。
    - 将添加剂（Additives）的 Y 轴起始点与渲染坐标对齐（从 `propY - 40` 调整为更精准的范围）。
    - 为搅拌棒的左右区域增加 10 像素的额外宽度缓冲。
- **预期效果**：点击“顺搅/逆搅”文字或杆部均能稳定触发。

### 4. 属性矩阵最后清理
- **文件**：`src/content/ingredients.ts`
- **修改点**：再次确认 `soda_water` 仅包含 `sparkle` 增量，且 `volume` 为 0（如果用户要求严格不改体积）。

## Verification Steps
1. **形状测试**：选择方波，进行顺/逆时针搅拌，观察波形是否仅做平移。
2. **反馈测试**：点击搅拌按钮，确认是否每次点击波形都会发生位移。
3. **命中测试**：点击搅拌工具的文字中心、边缘以及杆部，确认是否均能触发动作。
4. **编译验证**：运行 `npm run typecheck`。
