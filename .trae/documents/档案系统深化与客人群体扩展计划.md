# 档案系统深化与客人群体扩展计划

## Summary
本计划旨在落实 `cyber-resonance-bar-plan.md` 中的叙事支柱，通过深化“档案系统（Archive System）”和扩展“客人群体”，提升游戏的叙事深度和长线吸引力。玩家将通过调酒提升客人好感度（Affinity），解锁角色的背景故事（记忆碎片），并看到对话随关系拉近而产生的动态变化。

## Current State Analysis
- **数据结构简单**：目前 `GuestData` 仅包含基础信息和一套固定台词。
- **好感度系统孤立**：`guestAffinity` 虽然在 `GameState` 中存储并在 `npc-system.ts` 中更新，但尚未对游戏体验产生实质影响。
- **客人群体单薄**：目前仅有 3 位基础客人（老陈、Z3R0、雷队）。
- **档案界面缺失**：玩家无法查看已解锁的客人信息。

## Proposed Changes

### 1) 扩展客人数据模型 (`src/content/guests.ts`)
- **变更**：升级 `GuestData` 接口，增加 `bio`（基础简介）、`archives`（分级解锁的档案条目）和 `affinityDialogues`（好感度驱动的动态台词）。
- **细节**：
    ```typescript
    export interface GuestArchive {
      threshold: number; // 解锁所需的好感度
      title: string;     // 条目标题
      content: string;   // 解锁内容
    }
    export interface GuestData {
      // ... 现有字段
      bio: string;       // 玩家初次见面可见的简介
      archives: GuestArchive[];
      dialogues: {
        // ... 现有基础台词
        affinityLevels: {
          neutral: string[]; // 0-20 好感
          friendly: string[]; // 21-50 好感
          trusted: string[];  // 51-80 好感
          resonant: string[]; // 81+ 好感
        };
      };
    }
    ```

### 2) 扩展客人群体 (`src/content/guests.ts` & `src/ui/pixel-assets.ts`)
- **变更**：新增 4 位具有鲜明赛博职业和故事背景的 NPC。
- **新增角色清单**：
    1. **Sarah (执行官)**：外表冷酷的高级管理层，实则深陷职业倦怠，档案揭示其对旧时代的怀念。
    2. **Miku (地下偶像)**：与 AI 竞争的肉身偶像，性格活泼但波形极不稳定，档案揭示其作为“纯人类”的坚持。
    3. **Unit-7 (AI 哲学家)**：试图理解“灵魂”的旧型号机器人，追求完美的正弦波，档案探讨数字生命的本质。
    4. **Jax (街头浪人)**：退役飞行员，言语粗鲁但内心真诚，档案揭示其在企业战争中的往事。
- **变更**：在 `pixel-assets.ts` 中为新 NPC 绘制对应的像素点阵图。

### 3) 动态叙事逻辑升级 (`src/systems/npc/npc-system.ts`)
- **变更**：新增 `getGuestDialogue(state: GameState)` 函数。
- **逻辑**：根据 `state.guestAffinity[guestId]` 的数值，从对应角色的 `affinityLevels` 中抽取台词。
- **效果**：随着好感度提升，客人的语气会从生疏变得亲近或吐露心声。

### 4) HUD 档案面板实现 (`src/ui/hud-renderer.ts` & `src/styles.css`)
- **变更**：
    - 在对话面板增加“查看档案（PROFILE）”按钮。
    - 实现一个全屏或半屏的 `archive-modal` 覆盖层。
    - 展示客人的职业、好感度等级、基础简介。
    - 列表展示档案条目：已解锁的显示标题和内容，未解锁的显示 `[锁定 - 需要好感度 XX]`。
- **样式**：保持赛博像素风格，使用青绿与深红配色。

## Assumptions & Decisions
- **好感度阈值**：暂时设定为每 20-30 点提升一个关系等级。
- **解锁即时性**：好感度变动后，下一单或下一次对话即采用新台词，档案面板实时更新。
- **兼容性**：现有 3 位客人的数据将按新格式重写，确保存档数据平滑过渡。

## Verification Steps
1. **数据校验**：确认 `npm run typecheck` 通过，新接口无属性缺失。
2. **流程验证**：
    - 进入游戏，点击“接受订单”前确认能通过 UI 看到客人简介。
    - 模拟多次完美调酒提升好感度。
    - 检查对话内容是否随关系等级变化。
    - 打开档案面板，验证好感度达标后档案内容从“锁定”变为“可见”。
3. **视觉检查**：确认新 NPC 的像素形象渲染正确，无错位或溢出。
